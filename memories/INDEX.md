# memories/ — INDEX

Context preservation for coding agents. If a work session may end (or lose context) mid-task, write a memory file here so the next session can resume without re-deriving state. Read this index at the start of every session (step 2 in [AGENTS.md](../AGENTS.md)).

**File naming:** `<topic>__memory_<YYYY-MM-DD>.md`
Example: `phase-b-crypto-prototype__memory_2026-08-03.md`

**Each memory should contain:**

- **Task in progress** — what was being worked on and which `plannings/` step it maps to.
- **Current state** — what is done, what is half-done, exact stopping point.
- **Key context** — non-obvious findings, gotchas, decisions in flight (not yet in ARCHITECTURE.md/docs).
- **Next actions** — the precise next steps to resume.

**Lifecycle:** memories are disposable. Once the task completes and a worklog is written, delete the memory file (or mark it `(superseded)` here) — stale memories are worse than none. Durable facts belong in `docs/` or `ARCHITECTURE.md`, not here.
